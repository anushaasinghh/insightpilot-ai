# Definition of Done — InsightPilot AI

A feature is only considered "Done" when ALL of the following are true:

## Code
- [ ] Feature works as described in the user story
- [ ] No console errors or exceptions in normal use
- [ ] Code is commented where logic is non-obvious
- [ ] No hardcoded API keys or secrets
- [ ] Function has a docstring explaining what it does

## Testing
- [ ] Unit tests written for all utility functions
- [ ] Manual test cases executed and passed
- [ ] Edge cases tested (empty file, wrong format, huge file)
- [ ] Tested in Chrome and Firefox

## Product
- [ ] Matches acceptance criteria in the GitHub Issue
- [ ] UI labels and messages are clear to a non-technical user
- [ ] Error messages are helpful (not just "Error occurred")

## Process
- [ ] Code committed with a meaningful commit message
- [ ] GitHub Issue moved to "Done" column
- [ ] If a bug was fixed: root cause documented in issue comments
